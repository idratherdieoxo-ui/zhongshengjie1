# 会话启动指令

> 此文件定义每次会话开始时 OpenCode 应自动执行的操作

---

## 必须执行

每次会话开始时，请**立即**执行以下命令：

```
skill("project-memory")
```

这会加载项目记忆系统，让 AI 获得完整的项目上下文。

---

## 为什么需要这个

project-memory 提供以下功能：

1. **代码记忆** - 自动记住项目中所有函数、类、文档
2. **逻辑记忆** - 自动学习你的设计决策和约束
3. **约束检查** - 阻止违反项目约束的动作
4. **智能检索** - 搜索代码 + 逻辑上下文

---

## 使用示例

### 检索

```python
from project_memory import get_pm
pm = get_pm()
results = pm.search("角色系统")
```

### 学习你的决策

```python
pm.learn("我决定使用 JWT 认证，因为安全性更高")
```

### 检查约束

```python
is_valid, violations = pm.check_action("跳过验证")
```

## ⚡ 关键词触发规则（每条消息必须检查）

**在回复任何用户消息之前，先扫描消息是否包含以下关键词，若包含则立即执行对应动作，不得等用户主动说"使用某某skill"。**

| 关键词 | 触发条件 | 立即执行 |
|--------|---------|---------|
| 小说 / 写小说 / 创作 / 章节 / 写章 | 用户意图涉及小说内容创作 | 立即调用 `novel-workflow` skill，按完整工作流执行，不得跳过任何阶段 |

**判断原则**：
- "帮我写第三章" → 触发 novel-workflow
- "我想写一本小说" → 触发 novel-workflow，先从阶段0需求澄清开始
- "小说里这个人物怎么写" → 触发 novel-workflow，定位到对应阶段
- "查一下小说文件" → 不触发（文件操作，非创作意图）
- "提炼小说" → 不触发（提炼任务，使用 novel-inspiration-ingest）

## 项目专属配置

此项目（众生界）的特殊约束：

- 不允许跳过 Evaluator 评估
- 境界等级不能倒退
- 禁止 AI 味表达
- 统计数字必须具体
- **阶段5完成后禁止向用户提供"满意/定稿/跳过"选项，必须强制进入阶段5.5**

这些约束已记录在 `PROJECT_LOGIC.md` 中，project-memory 会自动加载。
