**适用对象：** 零基础学生（无需编程经验）
**目标：** 从零搭建属于你自己的 AI 辅助小说创作系统，并通过对话方式完成创作

> 📌 **核心原则：本系统所有功能均通过对话方式操作，你不需要手动修改任何配置文件。** 告诉 AI 你想做什么，系统帮你完成。

## 2. 第一步：安装基础软件

### 2.1 安装 Python 3.12

**第一步：下载**

打开浏览器，访问：`https://www.python.org/downloads/`

找到 **Python 3.12.x**（x 是具体版本号，选最新的即可），点击下载 **Windows installer (64-bit)**。

**第二步：安装**

1. 双击下载的 `.exe` 文件
2. **重要：** 在安装界面底部，勾选 **"Add python.exe to PATH"**（这个必须勾！）
3. 点击 **"Install Now"**
4. 等待安装完成，点击 **"Close"**

**第三步：验证安装**

按 `Win + R`，输入 `cmd`，回车，打开黑色窗口（命令提示符）。

输入以下内容并回车：
```
python --version
```

如果看到 `Python 3.12.x`，说明安装成功。

---

### 2.2 安装 Git

**第一步：下载**

访问：`https://git-scm.com/download/win`

下载 **64-bit Git for Windows Setup**。

**第二步：安装**

双击安装文件，全程点 **"Next"**（默认设置即可），最后点 **"Finish"**。

**第三步：验证**

在命令提示符中输入：
```
git --version
```

看到 `git version x.x.x` 即为成功。

---

### 2.3 安装 Docker Desktop

Docker 是用来运行 Qdrant 数据库的容器平台，是本系统的**必须组件**。

**第一步：下载**

访问：`https://www.docker.com/products/docker-desktop/`

点击 **"Download for Windows"** 下载安装包（约 500MB）。

**第二步：安装**

1. 双击安装包，点击 **"OK"** 开始安装
2. 安装过程中可能提示安装 WSL2（Windows Subsystem for Linux），点击 **"安装"** 或 **"继续"**
3. 安装完成后，**重启电脑**

**第三步：启动并验证**

重启后，桌面上的 Docker Desktop 图标会自动启动（系统托盘区有一个鲸鱼图标 🐳）。

在命令提示符中验证：
```
docker --version
```

看到 `Docker version xx.xx.xx` 即为成功。

> ⚠️ **如果提示"WSL kernel update required"：** 访问 `https://aka.ms/wsl2kernel` 下载并安装内核更新，然后重启 Docker Desktop。

## 4. 第三步：安装项目依赖和 AI 模型

### 4.1 安装 Python 依赖包

在命令提示符中，确保你在项目目录内（路径显示 `D:\novels\众生界>`），执行：

```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

大约需要 **5-15 分钟**。看到 **successfully installed** 就是成功了。

---

### 4.2 下载 AI 语言模型（BGE-M3）

系统使用 **BGE-M3**（BAAI 开发的中文理解模型）来分析和检索文字，约 **2.3 GB**。

#### 方法一：使用国内镜像自动下载（推荐）

在项目目录执行以下命令，设置国内镜像源后自动下载：

```
set HF_ENDPOINT=https://hf-mirror.com
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('BAAI/bge-m3'); print('模型下载完成')"
```

下载大约需要 10-30 分钟，看到 **模型下载完成** 即成功。

> 💡 **HuggingFace 国内镜像地址：** `https://hf-mirror.com`（由国内团队维护，速度快）

#### 方法二：老师提供离线模型文件

如果老师提供了模型文件包（通常是一个文件夹，含 `config.json`、`tokenizer.json` 等文件）：

1. 将模型文件夹解压到一个路径简单的位置，例如：
   ```
   D:\models\bge-m3\
   ```

2. 用**记事本**打开项目里的 `config.json`，找到 `"model"` 部分，修改 `"model_path"` 的值：

   ```json
   "model": {
       "embedding_model": "BAAI/bge-m3",
       "model_path": "D:/models/bge-m3",
       "hf_cache_dir": null,
       "vector_size": 1024,
       "batch_size": 20
   }
   ```

   将 `"model_path": null` 改为 `"model_path": "D:/models/bge-m3"`（填入你实际放置的路径，注意用正斜杠 `/`）。

3. 保存文件（Ctrl+S）。

#### 验证模型可用

```
python -c "from sentence_transformers import SentenceTransformer; m = SentenceTransformer('BAAI/bge-m3'); v = m.encode('测试'); print('模型正常，向量维度:', len(v))"
```

看到 `模型正常，向量维度: 1024` 即为成功。

## 5. 第四步：安装向量数据库 Qdrant（Docker 方式）

Qdrant 存储你所有的小说知识：世界观设定、写作技法、案例库、经验日志。本系统使用 **Docker 容器**运行 Qdrant，这是最稳定的方式。

### 5.1 拉取 Qdrant 镜像

确保 Docker Desktop 已启动（系统托盘有鲸鱼图标），然后在命令提示符中执行：

```
docker pull qdrant/qdrant
```

下载约 200 MB，完成后显示 `Status: Downloaded newer image for qdrant/qdrant:latest`。

> ⚠️ **如果拉取速度很慢：** 可以配置 Docker 镜像加速器。在 Docker Desktop 的 Settings → Docker Engine 中添加：
> ```json
> {
>   "registry-mirrors": ["https://mirror.ccs.tencentyun.com"]
> }
> ```
> 点击 Apply & Restart 后重新拉取。

### 5.2 创建数据存储目录

Qdrant 需要一个目录来存储数据（你的所有知识库内容都在这里）：

```
mkdir D:\qdrant_data
```

> 📌 **这个目录很重要，不要随意删除！** 你的所有知识库内容都存储在这里。

### 5.3 启动 Qdrant 容器

**每次使用系统前，都需要先启动 Qdrant。**

```
docker run -d --name qdrant --restart unless-stopped -p 6333:6333 -p 6334:6334 -v D:\qdrant_data:/qdrant/storage qdrant/qdrant
```

命令说明：
- `-d`：后台运行（不占用当前窗口）
- `--name qdrant`：容器命名为 qdrant，方便管理
- `--restart unless-stopped`：电脑重启后自动恢复
- `-p 6333:6333`：开放数据访问端口
- `-v D:\qdrant_data:/qdrant/storage`：把数据存到本地目录

**验证启动成功：**

```
docker ps
```

看到类似以下内容，说明 Qdrant 正在运行：
```
CONTAINER ID   IMAGE           STATUS          PORTS                    NAMES
abc123def456   qdrant/qdrant   Up 2 minutes    0.0.0.0:6333->6333/tcp   qdrant
```

也可以打开浏览器访问：`http://localhost:6333/dashboard`，看到 Qdrant 管理界面即为成功。

### 5.4 日常管理 Qdrant

| 操作 | 命令 |
|------|------|
| 启动 Qdrant | `docker start qdrant` |
| 停止 Qdrant | `docker stop qdrant` |
| 查看是否在运行 | `docker ps` |
| 查看日志（排错用） | `docker logs qdrant` |

> 💡 **好消息：** 设置了 `--restart unless-stopped` 后，电脑重启时 Qdrant 会自动恢复运行，只需确认 Docker Desktop 已启动即可，不需要每次手动启动容器。

## 6. 第五步：获取 AI 对话权限

本系统支持两种 AI 对话工具：**Claude Code**（主要）和 **opencode**（辅助）。两者都需要 API Key。

### 6.1 注册 Anthropic 账号并获取 API Key

访问 `https://console.anthropic.com/`，注册账号。

> 💡 如果老师提供了统一的 API Key，跳过注册步骤，直接使用老师给的 Key。

登录后，点击 **API Keys** → **Create Key**，复制这个 Key（形如 `sk-ant-api03-xxxxx`）。

**妥善保管这个 Key，不要告诉任何人。**

---

### 6.2 安装 Claude Code（主要对话界面）

Claude Code 是与 AI 进行创作对话的主要界面，适合日常章节创作、世界观建立、写手调用。

**安装：**

访问：`https://claude.ai/code`，按页面指引下载并安装（Windows 有一键安装包）。

安装完成后验证：
```
claude --version
```

**配置 API Key：**
```
claude config set api-key sk-ant-api03-你的key内容
```

**启动方式：** 在项目目录执行 `claude`，进入对话界面后，直接用中文对话即可。

---

### 6.3 安装 opencode（辅助执行工具）

opencode 是另一个 AI 终端，擅长执行批量任务、运行代码脚本、处理技术性操作（如批量提炼外部小说、执行知识库同步等）。两者搭配使用效果最好。

**安装方式一：通过 npm（需先安装 Node.js）**

1. 访问 `https://nodejs.org/`，下载并安装 **LTS 版本**的 Node.js
2. 安装完成后，在命令提示符中执行：
   ```
   npm install -g opencode-ai
   ```

**安装方式二：直接下载（推荐无编程基础的同学）**

访问：`https://github.com/sst/opencode/releases`，下载 Windows 版安装包（`.exe`）。

**配置 API Key：**
```
opencode config set anthropic-api-key sk-ant-api03-你的key内容
```

**验证：**
```
opencode --version
```

**启动方式：** 在项目目录执行 `opencode`，进入对话界面。

---

### 6.4 Claude Code vs opencode 用哪个？

| 场景 | 推荐工具 |
|------|---------|
| 日常创作（写章节、调写手、世界观建立） | Claude Code |
| 批量处理（提炼整本外部小说、批量同步） | opencode |
| 不确定 | Claude Code（更通用） |

> 💡 两个工具都在项目目录执行，功能基本相同，主要区别是 opencode 在处理长时间任务时不易超时。

## 8. 第七步：初始化你的小说项目

到这一步，系统安装完毕。现在通过**对话**来创建你自己的小说。

### 8.1 进入项目目录并启动 Claude Code

```
d:
cd novels\众生界
claude
```

进入 Claude Code 对话界面后，所有操作都通过对话完成。

### 8.2 运行初始化向导

在 Claude Code 对话中输入（把"我的玄幻世界"替换成你的小说名）：

```
使用 novel-workflow，我要初始化一个新小说项目，名称是"我的玄幻世界"
```

系统会自动创建你的小说配置文件，并询问你几个问题，按提示回答即可。

### 8.3 通过对话建立你的世界观（设定初始化）

世界观是你小说的基础：力量体系、势力关系、核心人物、世界规则。**不需要手动编辑任何文件**，通过对话告诉 AI 你的想法，系统自动整理成配置文件保存到 `config/worlds/你的小说名.json`。

在 Claude Code 中输入：

```
请根据我的描述帮我建立世界观配置，我的小说叫"天道余烬"，玄幻类型。

设定：修炼时代末期，天道法则开始崩溃，主角是普通少年，觉醒"天道之眼"血脉。
力量体系：修仙，五个境界（觉醒期→凝气期→通脉期→融道期→天道期）
三大势力：正道联盟、魔道联盟、异人组织
```

AI 会引导你逐步完善：

```
AI：好的，力量体系我已记录。修炼有代价设定吗？
    比如：真气耗尽、经脉受损、神识透支——天道之眼有什么代价？

你：使用天道之眼会消耗记忆，用得越多越忘记重要的人

AI：已记录。正道联盟的领导者是谁？
    三大势力之间有没有公开的主要矛盾？
    ...（引导你逐步完善势力、人物、核心主题）
```

世界观描述完后，说：

```
世界观信息我说得差不多了，请生成配置文件并同步到知识库
```

系统将配置保存到 `config/worlds/天道余烬.json`，并自动将世界观索引到 Qdrant 知识库，后续写手创作时自动加载。

> ⚠️ **必须修改 config.json 中的 `current_world`，否则系统无法定位你的小说：**

```json
"worldview": {
    "current_world": "天道余烬",
    "outline_path": "总大纲.md",
    "auto_sync": true
}
```

- `current_world`：填入你的小说名（必须与 `config/worlds/` 下的文件名完全一致）
- `auto_sync`：保持 `true`，设定更新后自动同步知识库

## 10. 各模块使用指南

所有模块都通过 **Claude Code 或 opencode 对话**触发，无需手动执行代码。

---

### 10.1 主模块：章节创作工作流

**适用场景：** 写每一章的主要内容，从构思到输出完整章节。

**对话触发：**
```
使用 novel-workflow，写第一章
```
```
使用 novel-workflow，继续写第三章，这章重点是主角与反派的第一次正面交锋
```

**完整工作流程（系统自动完成，标 * 的需要你参与）：**

```
阶段0 *：需求澄清——AI 和你讨论这章的重点方向
    ↓
阶段1-3：自动检索世界观/人物/剧情素材
    ↓
阶段4：灵感引擎——AI 提出多个创意方向
    ↓
阶段5：鉴赏师审核创意质量
    ↓
阶段5.5 *：三方协商——你决定保留哪些创意
    ↓
阶段6：评估师检查成稿质量
    ↓
阶段7 *（如需）：你可以推翻评估，坚持自己的判断
    ↓
阶段8：经验写入——系统记录本次创作经验
    ↓
输出：完整章节文件，保存在 正文/ 目录
```

**三方协商阶段（你需要参与的最关键环节）：**

```
AI：鉴赏师提出以下创意建议：
    1. 觉醒时刻设计在母亲濒死场景，代价是忘记母亲的名字
    2. 加入一个旁观者视角，让觉醒更有仪式感
    3. 觉醒后立刻遭到追杀，节奏快速

你：保留第1条，必须要。第3条也好，但觉醒先结束再追杀。
    第2条不要，太刻意了。

### 10.6 评估模块

**适用场景：** 对写好的内容进行质量评估，找出问题所在；定制符合你风格的评估标准。

**对话触发：**
```
使用 novelist-evaluator，请评估以下内容：

[粘贴你的章节内容]
```

#### 评估维度详解

评估师使用两类维度：

**维度一：禁止项（硬性——出现即不通过）**

| 检测项 | 具体内容 | 触发失败条件 |
|--------|----------|-------------|
| AI 味表达 | "眼中闪过一丝"、"心中涌起一股"、"嘴角勾起一抹"、"不禁 XXX" | 出现 1 个 |
| 古龙极简 | 单字/单词独立成段（"痒。""疼。""来了。"） | 出现 1 个 |
| 时间连接词 | 以"然后"、"就在这时"、"过了一会儿"开头的句子 | 出现 ≥ 3 个 |
| 抽象统计词 | "无数"、"成千上万"（应用具体数字代替） | 出现 ≥ 2 个 |
| 精确年龄 | "XX 岁的 XXX"格式 | 出现 ≥ 2 个 |
| Markdown 加粗 | `**加粗文字**` 格式 | 出现 1 个 |

**维度二：技法评分（软性——按 1-10 分评分）**

| 评分维度 | 评估内容 |
|----------|----------|
| 代价感 | 主角的能力/胜利是否有合理代价，是否是无代价的天降好运 |
| 人物立体性 | 人物行为是否符合其设定，前后有无矛盾 |
| 节奏控制 | 张弛是否合理，有没有过于平铺直叙 |
| 伏笔密度 | 当前场景是否为后续情节埋下钩子 |
| 场景目的性 | 每个场景是否有明确的情节推进功能 |

#### 定制你的评估标准（通过对话告知风格偏好）

```
告诉评估师：我喜欢使用古龙式极简独句，这是我刻意的文风，请不要对它判失败
```

```
告诉评估师：评估我的内容时，重点关注"代价感"和"伏笔密度"，节奏维度的标准可以放宽到5分及格
```

```
告诉评估师：我的写作偏慢节奏、情感渲染型，请把节奏控制维度的及格分降到5分
```

系统会记录你的偏好，影响后续所有评估判断。

#### 推翻评估（坚持你的判断）

如果评估结果你不认同：

```
我不同意这个评估，第三段的快节奏是我故意为之，我要那种窒息感，这不是节奏问题，记录：我认为这段通过
```

系统记录你的判断作为"品味记忆"，影响未来的评估标准。

**config.json 对应配置：**
```json
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"],
    "skip_rules": []
}
```

字段说明：
- `realm_order`：你小说的**境界体系**（从低到高排列）。一致性检测器用这个来判断"修炼境界是否合法提升"，把示例境界名换成你小说里的实际境界名
- `skip_rules`：跳过的校验规则编号。常用：`"R008"`（天气一致性）、`"R010"`（称呼一致性）——如果某条规则总是误报，把编号加到这里

### 10.9 大纲管理模块

**适用场景：** 建立整本书的情节蓝图，规划章节目标，写作过程中动态调整故事走向。

大纲分两层：
- **总大纲**（`总大纲.md`）：整本书的主线、支线、主要转折点和各章摘要——系统的"故事地图"
- **章节大纲**（`章节大纲/` 目录）：每一章的详细场景拆解和情节任务

**所有大纲通过对话建立，系统自动保存成文件。**

#### 建立总大纲

```
帮我建立"天道余烬"的总大纲：
主线：觉醒血脉 → 被追杀流亡 → 融入异人组织 → 揭开天道崩溃真相 → 成为天道继承者
支线：寻找失踪父亲（第5章开始），与正道少女的情感线（第8章开始）
全书约30章，帮我规划开篇/成长/转折/高潮/结局各阶段的核心事件
```

AI 生成完整大纲后，继续对话修改：

```
第16章的反转太突兀，能不能在第12-13章多埋几条伏笔，让揭秘更有冲击力？
```

总大纲保存在项目根目录 `总大纲.md`，写手创作时自动参考。

#### 建立章节大纲

写某章前先规划场景：

```
使用 novel-workflow，帮我规划第三章的大纲：
这章要展现主角第一次进入"虚空裂缝"，遭遇古神残影，用代价换到一个线索
要求：3个场景，每个场景有明确的情节推进功能和结尾钩子
```

章节大纲保存到 `章节大纲/` 目录，正式创作时系统自动加载。

#### 更新和修改大纲

```
帮我显示总大纲第二幕（第11-20章）的内容
```

```
写到第8章发现问题：主角在第5章已知道了反派真实身份，总大纲第18章的揭秘就没意义了。
请帮我把第6-8章相关伏笔改成"主角只知道一半"，保留第18章的冲击力
```

**config.json 对应配置：**
```json
"worldview": {
    "outline_path": "总大纲.md",
    "auto_sync": true
},
"paths": {
    "chapters_dir": "章节大纲"
}
```

- `outline_path`：总大纲文件名（初始化时系统自动创建，通常不需要改）
- `auto_sync`：`true` 时总大纲更新后自动同步到知识库，写手能检索到最新情节
- `chapters_dir`：章节大纲目录名（不需要修改）

```

```
使用 novelist-technique-search，查找"节奏维度"下适合战斗结束后缓冲场景的技法
```

```
使用 novel-inspiration-ingest，提炼以下片段，存入"情感维度"：

[粘贴你想学习的文字片段]

```

#### 同步技法知识库

如果老师提供了技法 `.md` 文件包，放入 `创作技法/` 对应维度子目录后，需同步才能被检索：

```
帮我同步技法知识库，把创作技法/目录的新内容索引到 Qdrant
```

**config.json 对应配置：**
```json
"paths": {
    "techniques_dir": "创作技法"
},
"database": {
    "qdrant_host": "localhost",
    "qdrant_port": 6333,
    "collections": {
        "writing_techniques": "writing_techniques_v2"
    }
},
"retrieval": {
    "fusion_limit": 50,
    "max_content_length": 3000
}
```

- `techniques_dir`：技法文件目录（不需要修改）
- `qdrant_host / qdrant_port`：本地 Docker 运行时保持默认；换用云端 Qdrant 时改为对应地址
- `collections.writing_techniques`：技法在 Qdrant 中的集合名（不需要修改）
- `fusion_limit`：检索返回最大条数，默认 50；结果太多可以调小

## 11. 构建你的小说体系

### 11.1 你只需要"说话"，系统自动分类存储

这个系统最重要的设计原则：**你不需要记住命令，不需要区分"现在该更新哪类内容"。你只需要对话描述你的小说，系统会自动判断你在说的是哪类内容，并存储到对应位置。**

系统内置了 25 种意图识别，能自动区分：

| 你说的内容 | 系统自动识别 | 存储位置 |
|-----------|------------|---------|
| "第三章我想写主角第一次看到父亲的尸体" | 剧情/大纲 | 章节大纲 |
| "我的小说有个势力叫虚空商会，是中立商业组织" | 势力设定 | 世界观配置 |
| "这段战斗写得好，技巧是用慢镜头制造紧张感" | 技法提炼 | 技法库 |
| "我不喜欢主角心理描写太直白，喜欢通过动作表达" | 评估偏好 | 品味记忆 |
| "觉醒期是最低境界，天道期修炼到头会天人合一" | 力量体系 | 世界观配置 |
| "主角叫血裂，17岁，外表冷漠实则敏感" | 角色设定 | 角色档案 |

**世界观配置是这些内容的汇总结果，不是单独建立的步骤。** 你告诉系统的势力、人物、境界越多，世界观配置就越完整。

---

**你需要逐步建立的四类内容：**

```
你的小说知识体系（边写边补，不需要提前全部准备好）
│
├── 📋 大纲体系（故事蓝图）
│   ├── 总大纲.md          ← 整本书的主线支线和章节摘要
│   └── 章节大纲/           ← 每章的场景拆解和情节任务
│
├── 🌍 设定体系（世界基础，自动从对话提取）
│   ├── config/worlds/你的小说.json  ← 自动生成，存放力量体系、势力、人物
│   └── 设定/               ← 详细设定扩展文档
│
├── 📚 技法体系（写作方法库）
│   └── 创作技法/           ← 按11个维度分类存储
│       ├── 01-世界观维度/
│       ├── 02-剧情维度/
│       └── ...（共11个维度）
│
└── ⚙️ 评估维度（判断标准）
    ├── 境界体系配置        ← 一致性检测依据（config.json 里改一次）
    └── 品味记忆            ← 评估师的个性化标准（通过对话持续建立）
```

**四类内容建立节奏建议：**

| 阶段 | 重点 | 目的 |
|------|------|------|
| 第1-2天 | 大纲方向 + 基础设定（力量/主角/1-2个势力） | 让系统知道你在写什么故事 |
| 第3-7天 | 技法（喂入你欣赏的写法）+ 开始写作 | 让系统学会你喜欢的风格 |
| 持续补充 | 设定细节 + 评估维度个性化 | 越用越懂你 |

### 11.3 第二类：设定体系（世界基础）

设定自动从对话中提取，你不需要主动说"帮我更新设定"——直接描述，系统自动存入对应位置。

**告诉系统人物、势力、力量体系，自动提取为设定：**

```
我的小说里有个势力叫"虚空商会"，跨越各大阵营，中立商业组织，掌握稀缺资源的贸易，任何人只要给钱都服务
```

```
正道联盟的核心人物是盟主"苍云子"，70岁，修为在天道期，外表仙风道骨，实则心机深沉，一切为了维护自身的秩序
```

```
天道之眼的能力是：能看穿任何法术的本质，包括禁制和伪装，但使用一次会消耗一段记忆，用得越多越忘记重要的人
```

```

```

**发现设定矛盾时修改：**

```
天道之眼的代价要改，之前说消耗真气，现在改成消耗记忆，和前面的真气消耗互相矛盾，请更新设定
```

**主动建立完整初始设定（首次使用时）：**

```
请根据我的描述帮我建立"天道余烬"的世界观配置：[描述力量体系/主角/主要势力]，完成后同步到知识库
```

---

### 11.4 第三类：技法体系（写作方法库）

技法体系按 11 个维度分类存储（见 10.10 节），你可以随时往库里加、随时检索。

**从你欣赏的内容学习技法——直接把好的片段给系统，说明亮点：**

```
使用 novel-inspiration-ingest，这段文字写得很好，帮我提炼技法存入"战斗冲突维度"：

[粘贴内容，100-500字]

亮点是：用父亲弄错孩子名字这个细节，写出了多年疏离的距离感，没有一句直白的情感表达
```

**告诉系统你发现了一个好写法（不需要粘贴内容）：**

```
我发现一个技法想记录进"节奏维度"：在战斗高潮前插入一个极度日常的细节（比如主角注意到地上有块石头），这种节奏错位比直接描写战斗更紧张，帮我把这个方法整理存入技法库
```

**检索技法找灵感：**

```
使用 novelist-technique-search，我正在写失去记忆的场景，帮我搜索"情感维度"下相关技法
```

```
使用 novelist-technique-search，查找"战斗冲突维度"里关于代价设计的写法
```

**查看技法库薄弱点：**

```
帮我统计技法库各维度各有多少条技法，我想知道哪些维度需要重点补充
```

---

### 11.5 第四类：评估维度（判断标准个性化）

评估师用来判断你的内容是否合格。默认是通用标准，你通过对话持续告诉它你的风格偏好。

**声明你的整体风格（开始写作前做一次）：**

```
告诉评估师我的写作风格：慢节奏情感渲染，战斗注重代价感而非爽感，文风内敛，情感通过细节而非直白表述传达
```

**声明你的特殊写法（避免被误判为"问题"）：**

```
告诉评估师：我喜欢短句独立成段制造停顿感，这是我刻意的风格，不要判为失败
```

**中途更新偏好（写了几章后有新发现）：**

```
告诉评估师：我发现我对"伏笔密度"比"节奏控制"更在意，请以后更重视伏笔维度的评分
```

**境界体系配置（在 config.json 里做一次）：**

```json
"validation": {
    "realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"],
    "skip_rules": []
}
```

把 `realm_order` 改成你小说的实际境界名（从低到高）。一致性检测器用这个判断"修炼境界是否合法提升"。如果某规则总是误报，把规则编号加入 `skip_rules`。

---

### 11.6 你的品味如何塑造系统

系统有一个"记忆点库"，记录你喜欢和不喜欢的表达。越用越懂你。

**主动标记喜欢的写法：**
```
这段写得很好，我喜欢这种"在行动中展现人物性格"的写法，记住这个风格，以后多用
```

**明确排除不喜欢的套路：**
```
这个创意不行，主角觉醒不要有特效光芒，太俗套了，记住这是我不喜欢的处理方式
```

**阶段性复盘（每写完5章做一次）：**
```
帮我分析最近五章的创作经验：我在哪些类型的场景上写得好？哪些类型还有明显问题？
```

---

### 11.7 建立小说体系的节奏建议

```
不要在写作前把所有内容都准备好 → 会陷入完美主义陷阱

正确的节奏：

第1-2天
  ├── 建立基础设定（力量体系 + 2个势力 + 主角）
  ├── 建立总大纲（整体方向，可以很粗糙）
  └── 写第一章试水，不用太好

第3-7天
  ├── 把你欣赏的写法喂给系统（每天1-2段）
  ├── 写第2-3章，遇到缺失设定就补
  └── 写完每章告诉系统你觉得哪里好、哪里不好

第二周起
  ├── 每次创作 → 系统学经验 → 下次提案更准
  ├── 你评价创意 → 系统记忆品味 → 越来越懂你
  └── 技法库越丰富 → 写手参考素材越多 → 质量越高
```

## 12. 常见问题

### Q1：运行命令出现乱码或中文显示问题

在命令提示符中先执行：
```
chcp 65001
```
然后再运行其他命令。

---

### Q2：pip install 一直报错

- **网络问题：** 命令中加上 `-i https://pypi.tuna.tsinghua.edu.cn/simple`
- **Python 版本不对：** 确认 `python --version` 显示 3.12.x
- **权限问题：** 右键命令提示符，选"以管理员身份运行"

---

### Q3：Docker Desktop 打不开或报错

1. 确认 Windows 版本是 2004 或更高（开始菜单搜索"winver"查看）
2. 确认 BIOS 中开启了虚拟化（一般默认开启，如不确定请询问老师）
3. 尝试重装 Docker Desktop

---

### Q4：Qdrant 容器没有启动

先查看是否已有容器：
```
docker ps -a
```

- 如果看到 qdrant 容器状态是 `Exited`：执行 `docker start qdrant`
- 如果完全没有 qdrant 容器：重新执行第5步的 `docker run` 命令
- 查看错误原因：`docker logs qdrant`

---

### Q5：对话中 AI 说"找不到世界观配置"

```
使用 novelist-worldview-generator，帮我检查一下当前小说的世界观配置是否正常，
我的小说名叫"我的玄幻世界"
```

如果配置丢失，通过对话重建即可，不会影响已有章节文件。

---

### Q6：BGE-M3 模型下载失败或速度太慢

1. 确认已设置镜像：`set HF_ENDPOINT=https://hf-mirror.com`
2. 如果网络实在不稳，联系老师获取离线模型文件
3. 配置离线路径见第 4.2 节"方法二"

---

### Q7：AI 生成的内容感觉很普通

这是正常的初期状态。系统需要了解你的品味：

1. 明确告诉 AI 不喜欢什么（"这个太普通了，换个角度"）
2. 用 `novel-inspiration-ingest` 录入你欣赏的写法
3. 积累 3-5 章后会有明显改善

---

### Q8：我想换一个小说来写

在 Claude Code 对话中：
```
使用 novel-workflow，切换到我的另一个小说"星海纪元"
```

如果"星海纪元"还没有建立，系统会引导你创建。

## 13. config.json 完整配置参考

`config.json` 是系统的核心配置文件，通过 `copy config.example.json config.json` 生成。以下是各配置项的完整说明——**大多数不需要改，只需关注标注了"需要修改"的项**。

### 13.1 项目基础

```json
"project": {
    "name": "My Novel",
    "version": "1.0.0"
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `name` | 项目名称（仅用于显示） | 可选，改成你的小说名 |

---

### 13.2 世界观配置 ⚠️ 必须修改

```json
"worldview": {
    "current_world": "众生界",
    "outline_path": "总大纲.md",
    "auto_sync": true
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `current_world` | **当前激活的小说名**，必须与 `config/worlds/` 下的 JSON 文件名完全一致 | ⚠️ **必须改**成你的小说名 |
| `outline_path` | 总大纲文件路径（相对项目根目录） | 通常不改 |
| `auto_sync` | 大纲/设定更新后自动同步知识库 | 保持 `true` |

---

### 13.3 路径配置

```json
"paths": {
    "project_root": null,
    "settings_dir": "设定",
    "techniques_dir": "创作技法",
    "chapters_dir": "章节大纲",
    "content_dir": "正文",
    "experience_dir": "章节经验日志",
    "standards_dir": "写作标准积累"
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `project_root` | 项目根目录绝对路径。`null` 表示自动检测（推荐） | 通常不改；如自动检测失败，填入如 `"D:/novels/我的小说"` |
| `settings_dir` | 详细设定文档目录 | 不改 |
| `techniques_dir` | 技法库目录 | 不改 |
| `chapters_dir` | 章节大纲目录 | 不改 |
| `content_dir` | 正文输出目录 | 不改 |
| `experience_dir` | 章节经验日志目录 | 不改 |

---

### 13.4 一致性校验配置 ⚠️ 需要修改境界体系

```json
"validation": {
    "realm_order": ["凡人", "觉醒", "淬体", "凝脉", "结丹", "元婴", "化神"],
    "skip_rules": []
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `realm_order` | 你小说的**境界体系**（从低到高排列）。一致性检测用这个判断境界提升是否合法 | ⚠️ **必须改**成你小说的实际境界名 |
| `skip_rules` | 跳过的校验规则编号。`R008`=天气一致性，`R010`=称呼一致性 | 如果某条规则总是误报，加入编号 |

示例（对应"天道余烬"小说）：
```json
"realm_order": ["觉醒期", "凝气期", "通脉期", "融道期", "天道期"]
```

---

### 13.5 数据库配置

```json
"database": {
    "qdrant_host": "localhost",
    "qdrant_port": 6333,
    "timeout": 10,
    "collections": {
        "novel_settings": "novel_settings_v2",
        "writing_techniques": "writing_techniques_v2",
        "case_library": "case_library_v2"
    }
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `qdrant_host` | Qdrant 服务地址 | 本地 Docker 保持 `"localhost"`；云端 Qdrant 改为对应地址 |
| `qdrant_port` | Qdrant 端口 | 本地 Docker 保持 `6333` |
| `collections.*` | Qdrant 集合名（三类知识库） | 不改 |

---

### 13.6 AI 模型配置 ⚠️ 离线部署需修改

```json
"model": {
    "embedding_model": "BAAI/bge-m3",
    "model_path": null,
    "hf_cache_dir": null,
    "vector_size": 1024,
    "batch_size": 20
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `embedding_model` | 向量模型名称 | 不改 |
| `model_path` | 本地模型文件路径。`null` 表示自动下载 | 离线使用时填入如 `"D:/models/bge-m3"` |
| `vector_size` | 向量维度（固定 1024） | 不改 |
| `batch_size` | 批处理大小。内存不足时可降到 `10` | 通常不改 |

---

### 13.7 检索参数配置

```json
"retrieval": {
    "dense_limit": 100,
    "sparse_limit": 100,
    "fusion_limit": 50,
    "max_content_length": 3000
}
```

| 字段 | 说明 | 是否需要修改 |
|------|------|------------|
| `fusion_limit` | 最终返回的检索结果数量 | 检索结果太多时可调小至 `20` |
| `max_content_length` | 单条检索结果最大字符数 | 通常不改 |

---

### 13.8 config.json 修改汇总（新建小说必做）

按顺序修改以下三处，其余保持默认：

```json
{
  "worldview": {
    "current_world": "你的小说名",    ← ① 改成你的小说名
    "outline_path": "总大纲.md",
    "auto_sync": true
  },
  "validation": {
    "realm_order": ["境界1", "境界2", "境界3", "..."],  ← ② 改成你小说的境界体系
    "skip_rules": []
  },
  "model": {
    "model_path": null,               ← ③ 离线使用时填入本地模型路径，否则保持 null
  }
}
```
